Performance testing

[What is it? What we can test? When we shall it do?]

What metrics we shall measure?

  • Capacity (max throughput): RPS, VU
  • Response time: Average response time (percentiles)
  • Stability: Average uptime (before reboot), startup time
  • Availability: Average uptime (percent of time alive), 5** errors count
  • Scalability: scale factor (increase performance per new instance)
  • Critical resource (cpu/io)

What kind of load exists?

In this part we need to answer on a simple question: how users will react on performance degradation?

  • Open (multiple users) - new user will sent new requests.
  • Close (few users) - user will wait server response until new request will be sent.

Stateless or Stateful

  • Stateless - does not contain session inside service. Example: http, dns
  • Stateful - contain session inside service. Example: ftp, imap, pop3

Expected SLA

  • Expected load: average / peak / reliability factor
  • Expected response time
  • Expected reboot time
